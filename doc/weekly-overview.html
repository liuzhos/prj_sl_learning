第 1 周 · 工具链与前后端「先跑通」

目标
  • 装好编辑器、Git、Node、PHP、Composer；能在本地提交代码。
  • TypeScript 写最小脚本；React + TS 页面能运行并有简单交互。
  • Laravel 起一个返回 JSON 的接口；React 通过代理调用；初步验证 Sanctum 保护路由返回 401。
学习内容
  • 终端与版本号；Git 提交概念；npm、Vite、Laravel 路由与 JSON 响应。
  • 前后端分离下如何用开发代理避免 CORS；HTTP 状态码与 JSON 结构的基本直觉。
  • Sanctum 是什么、为何未授权会得到 401（先机制、后完整登录流程）。
参考资料
  • Visual Studio Code
  • GitHub Desktop
  • TypeScript Handbook
  • Vite 指南 · React Learn
  • Laravel Routing · Responses
  • Laravel Sanctum

第 2 周 · Sanctum 登录与受保护接口

目标
  • 走通「注册/登录 → 拿到可用凭证 → 请求受保护 API」的最小闭环（以实际采用的 Sanctum 模式为准：SPA Cookie 或 Personal Access Token）。
  • React 侧能带上凭证、处理 401/403 与登出后的状态。
学习内容
  • Laravel 用户模型、认证配置与 Sanctum 中间件；API 返回结构统一（成功、校验错误、未授权）。
  • 前端安全常识：不把密钥写进仓库；Token/Cookie 的存放与请求头惯例。
  • TypeScript：为接口响应定义类型，减少「字段名写错」。
参考资料
  • Sanctum（SPA API / Token）
  • Laravel Authentication
  • HTTP 状态码 · MDN

第 3 周 · Laravel API：校验、资源、分页

目标
  • 能读懂并仿写常见 CRUD 接口：列表（含分页/筛选骨架）、详情、创建、更新。
  • 表单校验与 422 错误体与前端字段一一对应。
学习内容
  • 控制器职责；Request 校验；API Resource（或统一数组结构）用于规范化 JSON。
  • Eloquent 查询、`paginate`、排序参数约定。
参考资料
  • Validation
  • Eloquent
  • Pagination
  • Eloquent API Resources

第 4 周 · React：列表、表单与接口约定

目标
  • 列表页：加载中、空状态、错误展示；分页控件与 query 参数同步（若项目用路由）。
  • 表单：受控输入、提交、展示服务端校验错误。
学习内容
  • React Hooks（`useState` / `useEffect`）；与项目一致的 HTTP 封装（如 axios 实例）。
  • React Router 基础；可选：公司 UI 组件库的一个列表+表单范式。
参考资料
  • React Learn
  • React Router
  • Axios 文档（若团队使用）

第 5 周 · 数据层：迁移、Eloquent、简单关联

目标
  • 为一个小业务加字段/表：写迁移、跑迁移、模型与 `$fillable` 等基本配置。
  • 读得懂一对一/一对多中的「一方/多方」与 `with` 预加载思路。
学习内容
  • Migrations；数据库约束与回滚意识；避免把敏感逻辑堆在控制器里（按团队分层习惯）。
  • 简单 N+1 问题概念：列表接口为何要用预加载。
参考资料
  • Migrations
  • Eloquent Relationships

第 6 周 · H5 / WebView 与工程习惯

目标
  • 理解嵌入 App 的 H5 常见差异：视口、安全区、键盘顶起、部分 WebView 行为。
  • 能按团队规范做环境变量、接口 baseURL、构建产物基本认知。
学习内容
  • 移动端 CSS 直觉；何时该和后端/原生同事对齐方案。
  • `.env` 与「不要把密钥提交进 Git」；前端构建 `npm run build` 在团队流水线中的位置（概念即可）。
参考资料
  • viewport meta · MDN
  • Vite 环境与变量

第 7 周 · 与团队仓库对齐：分支、MR、环境

目标
  • 能克隆公司项目、按文档起本地环境；在功能分支上开发并提交 MR/PR。
  • 读懂 Code Review 中「安全、风格、业务」三类意见的常见含义。
学习内容
  • 分支命名、提交信息习惯；解决简单合并冲突。
  • 项目内 README、接口文档、调试入口（如日志位置）的查找方法。
参考资料
  • GitHub · Using Git
  • Laravel Logging

第 8 周 · 小模块竖切与复盘

目标
  • 在带教人 Review 下，独立完成一个「简单资源」的竖切：API + React 页面（或其中一段由团队分工但自己端到端理解）。
  • 写简短自测清单与上线注意点（对接真实协作）。
学习内容
  • 任务拆分：接口字段 → 页面状态 → 边界情况（未登录、空列表、网络失败）。
  • 复盘：本周最常卡住的点与可沉淀的「团队一页纸」笔记。
参考资料
  • 以公司内部：接口约定、组件库文档、部署说明为准（外链仅作通用补充）。

若节奏更慢：可将每周拆成两周执行；第 1 周与「week1-plan」逐日版可对照使用。

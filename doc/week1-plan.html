第 6 天：React 调用 Laravel API（loading/error）

1, 目标
React 页面点击或自动请求 `GET /api/hello`；展示后端返回数据；处理加载/错误态；避免跨域坑。
2, 学习内容
React 的 `fetch`；用 Vite proxy 让 `/api` 转发到 Laravel，从而避免 CORS。
3, 参考资料
Vite Dev Server Proxy
MDN fetch
4, 练习
  1. 在 React 项目里配置 Vite 代理（核心是 `server.proxy`）： 
    // 打开 vite.config.ts
// 添加/修改为下面这种结构（按项目文件命名调整）：
server: {
  proxy: {
    "/api": "http://127.0.0.1:8000",
  },
},
    如果不知道 `vite.config.ts` 在哪，就在 VS Code 搜索 `vite.config`。
  2. 修改 `src/App.tsx`：自动请求并显示 message/time（示例直接粘贴即可跑）： 
    import { useEffect, useState } from "react";

type HelloResponse = {
  message: string;
  time: string;
};

export default function App() {
  const [loading, setLoading] = useState(true);
  const [data, setData] = useState<HelloResponse | null>(null);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const run = async () => {
      try {
        setLoading(true);
        setError(null);

        const res = await fetch("/api/hello");
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        const json = (await res.json()) as HelloResponse;
        setData(json);
      } catch (e) {
        setError(e instanceof Error ? e.message : "unknown error");
      } finally {
        setLoading(false);
      }
    };

    run();
  }, []);

  return (
    <div style={{ padding: 24, fontFamily: "system-ui" }}>
      <h1>Week1 API Call</h1>
      {loading && <p>Loading...</p>}
      {error && <p style={{ color: "crimson" }}>Error: {error}</p>}
      {data && (
        <div>
          <p>Message: {data.message}</p>
          <p>Time: {data.time}</p>
        </div>
      )}
    </div>
  );
}
    同样注意：HTML 转义只是为了显示，这里最终文件要用正常 JSX/TS 符号。
  3. 同时运行后端：`php artisan serve`；前端：`npm run dev`。
  4. 浏览器确认页面能显示 `message/time`。
完成标准
页面显示从 Laravel 拉到的 message/time，且能看到 loading/error。

第 7 天：Sanctum 未登录访问受保护接口返回 401

1, 目标
安装 Sanctum；加一个 `auth:sanctum` 受保护路由；未带 token 访问得到 401（最直观验证认证机制）。
2, 学习内容
理解 `auth:sanctum` 的作用：没凭证进不去。
3, 参考资料
Laravel Sanctum
4, 练习（做验证即可，不需要完整登录流程）
  1. 安装 Sanctum： 
    cd 'C:\learning\week1\laravel-api'
composer require laravel/sanctum
  2. 在 `routes/api.php` 加受保护路由（放底部）： 
    use Illuminate\Http\Request;

Route::get('/protected', function (Request $request) {
    return response()->json([
        "ok" => true,
        "user" => $request->user(),
    ]);
})->middleware('auth:sanctum');
  3. 用 curl 测未授权访问是否是 401：
  4. curl -i http://127.0.0.1:8000/api/protected
    观察返回头：应出现 `401 Unauthorized`。
  5. 记录产出到 `day7.md`：
  6. Set-Content -Path 'C:\learning\week1\day7.md' -Value "day7: /api/protected should be 401 without token"
完成标准
未登录访问 `/api/protected` 得到 `401 Unauthorized`。

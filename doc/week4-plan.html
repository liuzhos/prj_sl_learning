第 4 周学习计划（React：列表、表单与接口约定）

目标：把后端 API 真实接到前端页面，形成可用的列表与表单基础范式。

目录

  • 第 1 天：页面结构与路由骨架
  • 第 2 天：列表页加载/空态/错误态
  • 第 3 天：分页与查询参数同步
  • 第 4 天：创建表单与字段校验展示
  • 第 5 天：编辑表单与回填
  • 第 6 天：删除与二次确认
  • 第 7 天：前端回归与重构整理

第 1 天：页面结构与路由骨架

目标
搭建列表/表单/详情页面骨架，明确页面状态位置。
学习内容
路由拆分、页面级状态与组件级状态分工。
参考资料
  • React Router
练习
  1. 建 3 个页面并接好路由。
  2. 每页放占位组件，确保跳转可用。

第 2 天：列表页加载/空态/错误态

目标
列表页具备可用的状态反馈，不再只看控制台错误。
学习内容
Loading/Empty/Error 的最小 UI 规范与切换逻辑。
参考资料
  • 条件渲染
练习
  1. 实现加载中骨架。
  2. 空数据和错误提示可区分显示。

第 3 天：分页与查询参数同步

目标
分页状态可和 URL 同步，刷新后能恢复当前视图。
学习内容
查询参数读取、写入与防抖更新。
参考资料
  • URLSearchParams
练习
  1. 切页时更新 `page` 参数。
  2. 刷新页面后自动恢复页码。

第 4 天：创建表单与字段校验展示

目标
创建表单可提交，422 错误可映射到具体字段。
学习内容
受控输入、提交中禁用、防重复提交。
参考资料
  • Managing State
练习
  1. 完成创建表单提交。
  2. 显示后端字段错误信息。

第 5 天：编辑表单与回填

目标
编辑页可回填并保存更新，保持与创建页一致的交互。
学习内容
详情加载、初始值设置、更新成功反馈。
参考资料
  • Synchronizing with Effects
练习
  1. 加载详情后回填表单。
  2. 保存后回列表并显示提示。

第 6 天：删除与二次确认

目标
删除行为有明确确认与结果反馈，避免误操作。
学习内容
危险操作确认、删除后列表刷新策略。
参考资料
  • confirm
练习
  1. 实现删除确认弹窗。
  2. 删除后自动刷新并提示成功。

第 7 天：前端回归与重构整理

目标
完成“列表-创建-编辑-删除”全链路回归并沉淀可复用代码。
学习内容
抽取 API 层、复用状态组件、形成页面模板。
参考资料
  • React Learn
练习
  1. 整理回归清单并逐条勾选。
  2. 写 `week4-retro.md` 记录改进点。

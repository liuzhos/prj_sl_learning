第 8 周学习计划（小模块竖切与复盘）

目标：独立完成一个小模块端到端交付，并形成结构化复盘。

目录

  • 第 1 天：选题与边界确认
  • 第 2 天：接口设计与数据结构
  • 第 3 天：后端实现与自测
  • 第 4 天：前端页面实现
  • 第 5 天：联调与问题清单
  • 第 6 天：演示准备与自测报告
  • 第 7 天：最终复盘

第 1 天：选题与边界确认

目标
明确模块范围，避免任务失控。
学习内容
需求拆解、范围管理、MVP 思维。
参考资料
  • User Stories
练习
  1. 写模块边界（做什么/不做什么）。
  2. 列出 5 条验收标准。

第 2 天：接口设计与数据结构

目标
先完成接口契约，再开始编码。
学习内容
字段类型、状态码、错误结构一致性。
参考资料
  • Controllers
练习
  1. 写接口草案文档。
  2. 与前端字段约定对齐。

第 3 天：后端实现与自测

目标
后端实现最小 CRUD 并通过自测。
学习内容
校验、资源输出、错误处理。
参考资料
  • Validation
  • API Resources
练习
  1. 实现核心接口。
  2. 补 3 个异常输入测试。

第 4 天：前端页面实现

目标
完成列表与表单页面，具备基本交互与反馈。
学习内容
数据请求、状态渲染、表单提交反馈。
参考资料
  • React Learn
练习
  1. 接入列表查询接口。
  2. 完成创建/编辑流程。

第 5 天：联调与问题清单

目标
打通前后端链路，修复主要阻塞问题。
学习内容
联调顺序、接口对齐、错误归因。
参考资料
  • HTTP 状态码
练习
  1. 列问题清单并逐条关闭。
  2. 记录修复前后差异。

第 6 天：演示准备与自测报告

目标
形成可演示版本，并准备测试证明材料。
学习内容
演示路径设计、测试清单化、风险提示。
参考资料
  • Smoke Test
练习
  1. 准备 5 分钟演示脚本。
  2. 输出正常/异常/边界测试清单。

第 7 天：最终复盘

目标
总结本阶段能力成长与下阶段改进方向。
学习内容
复盘框架：目标、结果、问题、改进、行动。
参考资料
  • Retrospective
练习
  1. 写 `week8-retro.md`。
  2. 列出下阶段 3 个可执行优化项。

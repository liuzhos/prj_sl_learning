第 7 周学习计划（与团队仓库对齐：分支、MR、环境）

目标：从“个人开发”过渡到“团队协作可交付”。

目录

  • 第 1 天：克隆项目与本地启动
  • 第 2 天：分支策略与命名规范
  • 第 3 天：提交信息与原子提交
  • 第 4 天：发起 MR/PR 与描述模板
  • 第 5 天：处理 Review 意见
  • 第 6 天：简单冲突解决
  • 第 7 天：协作流程复盘

第 1 天：克隆项目与本地启动

目标
按团队文档完整拉起项目。
学习内容
项目结构阅读、启动依赖检查。
参考资料
  • Cloning a Repository
练习
  1. 本地启动前后端。
  2. 记录首次启动问题清单。

第 2 天：分支策略与命名规范

目标
形成“功能分支开发”习惯，不在主分支直接改。
学习内容
feature/hotfix 分支语义、命名规范。
参考资料
  • Gitflow Overview
练习
  1. 创建 feature 分支并提交 1 次改动。
  2. 同步主分支最新代码。

第 3 天：提交信息与原子提交

目标
每次提交表达单一意图，方便 Review 与回滚。
学习内容
提交粒度、提交信息结构、避免“巨大提交”。
参考资料
  • Conventional Commits
练习
  1. 拆成 2~3 个原子提交。
  2. 每条 commit message 写清意图。

第 4 天：发起 MR/PR 与描述模板

目标
提交可审阅的 MR/PR，包含变更说明与测试计划。
学习内容
PR 模板字段：背景、改动、风险、验证。
参考资料
  • GitHub Pull Requests
练习
  1. 发起一次 MR/PR。
  2. 补截图或接口请求示例。

第 5 天：处理 Review 意见

目标
高质量响应 Review，区分必须改与可讨论项。
学习内容
安全问题、逻辑问题、风格建议的优先级。
参考资料
  • Code Review Practices
练习
  1. 逐条回复 Review 评论。
  2. 补提交并更新 PR 描述。

第 6 天：简单冲突解决

目标
能处理常见文本冲突并确保逻辑不回退。
学习内容
冲突标记含义、手工合并、合并后回归。
参考资料
  • Resolving Merge Conflicts
练习
  1. 模拟同文件冲突并解决。
  2. 重新跑关键功能自测。

第 7 天：协作流程复盘

目标
完成从分支开发到合并的全流程演练。
学习内容
协作瓶颈识别、沟通前置与风险提示。
参考资料
  • Using Git
练习
  1. 输出“协作流程清单”一页纸。
  2. 写 `week7-retro.md`。

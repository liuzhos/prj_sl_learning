第 3 周学习计划（Laravel API：校验、资源、分页）

目标：把 API 从“能返回数据”升级到“结构稳定、错误清晰、列表可分页”。

目录

  • 第 1 天：需求拆解与接口字段约定
  • 第 2 天：创建接口 + Request 校验
  • 第 3 天：更新接口 + 局部更新规则
  • 第 4 天：列表接口 + 分页
  • 第 5 天：详情接口 + 资源转换
  • 第 6 天：筛选/排序参数约定
  • 第 7 天：接口回归测试与文档整理
建议：固定一个示例资源（如 posts/devices），整周围绕它迭代。

第 1 天：需求拆解与接口字段约定

目标
明确一个资源的字段、约束、返回结构，为后续 6 天开发打底。
学习内容
接口契约思维：字段命名一致、状态码明确、错误体可读。
参考资料
  • HTTP 状态码 · MDN
  • Laravel Responses
练习
  1. 写 `week3-api-spec.md`，包含字段表与示例 JSON。
  2. 约定统一返回壳：`ok/data/message/meta`。

第 2 天：创建接口 + Request 校验

目标
实现创建接口，成功返回 201，校验失败返回 422。
学习内容
Form Request 校验、错误字段映射、创建成功返回体规范。
参考资料
  • Validation
  • Controllers
练习
  1. 实现 `POST /api/{resource}`。
  2. 准备 1 组合法输入与 1 组非法输入做验证。

第 3 天：更新接口 + 局部更新规则

目标
实现 `PUT/PATCH`，并明确未传字段是否保留原值。
学习内容
幂等更新、部分更新、唯一字段更新时的校验例外写法。
参考资料
  • Validation unique
  • Eloquent
练习
  1. 实现 `PATCH /api/{resource}/{id}`。
  2. 新增“只改一个字段”的测试样例。

第 4 天：列表接口 + 分页

目标
实现可分页列表，返回 `data + meta`。
学习内容
`paginate` 参数处理、默认每页数、防止过大分页请求。
参考资料
  • Pagination
  • Query Builder
练习
  1. 实现 `GET /api/{resource}?page=1&per_page=10`。
  2. 补充 `meta.total/current_page/per_page`。

第 5 天：详情接口 + 资源转换

目标
实现详情接口并使用 Resource 统一输出格式。
学习内容
API Resource 的字段裁剪、时间格式统一、空值处理。
参考资料
  • Eloquent Resources
  • Helpers
练习
  1. 实现 `GET /api/{resource}/{id}`。
  2. 把创建、详情、列表三处输出结构对齐。

第 6 天：筛选/排序参数约定

目标
支持至少 1 个筛选和 1 个排序参数。
学习内容
查询参数白名单、非法参数处理、默认排序策略。
参考资料
  • Where Clauses
  • Ordering
练习
  1. 实现 `status` 筛选和 `sort` 排序。
  2. 写清非法参数时返回默认行为或错误信息。

第 7 天：接口回归测试与文档整理

目标
完成 CRUD + 分页 + 筛选 + 错误场景回归。
学习内容
接口测试清单化、可复测思维、面向协作的最小文档。
参考资料
  • HTTP Tests
  • Logging
练习
  1. 整理 Postman Collection 或 curl 脚本。
  2. 补 `week3-retro.md`（3 个收获 + 2 个待改进点）。
